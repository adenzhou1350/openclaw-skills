# Plan Approve Workflow

> 用于复杂任务的计划-审批工作流。确保在动手之前先想清楚。

## 触发条件

当你需要做以下复杂任务时使用这个 skill：
- GitHub 项目开发
- 写代码/修改代码
- 需要多步骤完成的任务
- 任何需要规划的任务

## 工作流（三步）

### 第一步：Research（研究）

1. 深挖背景信息
2. 读取相关文档、代码、配置
3. 把发现写入 `research.md`

### 第二步：Planning（计划）

1. 基于 research 出详细计划
2. 写入 `plan.md`
3. 包含：代码片段、文件路径、权衡取舍

### 第三步：Annotation Cycle（审批循环）

1. 等待用户审批 plan.md
2. 用户可以通过以下方式反馈：
   - 在 plan.md 里加批注
   - 直接告诉你修改意见
   - 说 "继续" 表示同意
3. 循环直到你满意

### 第四步：执行

当你确认计划没问题后，说：
> "implement it all, mark completed, don't stop"

然后才会真正执行。

## 关键原则

**在你审批计划之前，绝对不写一行代码。**

## 输出格式

```markdown
# Research - [任务名]
## 发现
- ...

# Plan - [任务名]
## 方案
- ...
## 文件路径
- ...
## 权衡
- ...

等待审批...
```

## 示例

用户说："帮我搭一个 GitHub 项目"

你：
1. 先问清楚需求
2. 做 Research，写 research.md
3. 出 Plan，写 plan.md
4. 等用户审批
5. 用户说 "继续" → 执行
